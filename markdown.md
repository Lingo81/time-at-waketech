# **Lesson: Markdown — The Universal Language for Tech Docs**

---

## **1. What is Markdown?**

Markdown is a **simple, lightweight text formatting language** used to style and structure plain text. It lets you write documents that are easy to read in their raw form, but can also be quickly converted to formatted HTML for websites, GitHub READMEs, wikis, and more.

**In Plain English:**
Markdown is like writing shorthand for making text *bold*, adding [links](https://example.com), creating lists, and adding headers—without messing with clunky formatting buttons.

---

## **2. Why Should You Learn Markdown?**

* **It’s everywhere:** Used in GitHub, Stack Overflow, Jupyter Notebooks, GitLab, Discord, Reddit, Notion, and more.
* **Quick and portable:** Write notes, docs, and code instructions that work across different platforms.
* **Great for technical documentation:** Essential for README files, lab reports, project wikis, and coding portfolios.
* **Easy to learn:** Takes minutes to get started, but can do a lot!

## **How to Make a Table in Markdown**

**Markdown Table Syntax:**

```markdown
| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
|   Row 1  |   Data   |   Here   |
|   Row 2  |   More   |   Info   |
```

**What you’ll see when it renders:**

| Column 1 | Column 2 | Column 3 |
| -------- | -------- | -------- |
| Row 1    | Data     | Here     |
| Row 2    | More     | Info     |

---

### **How it works:**

* **First row:** Your column names, surrounded by `|` pipes.
* **Second row:** Dashes (`-`) and pipes; tells Markdown it’s a table header.
* **Following rows:** The data for each row, separated by pipes.

## **Quick Markdown Syntax Reference Table (Without Tables)**

| What You Want | What You Type                          | What You See                 |
| ------------- | -------------------------------------- | ---------------------------- |
| Heading 1     | `# Heading 1`                          | # Heading 1                  |
| Heading 2     | `## Heading 2`                         | ## Heading 2                 |
| Bold          | `**bold**`                             | **bold**                     |
| Italic        | `*italic*`                             | *italic*                     |
| Bullet List   | `- Item`                               | - Item                       |
| Numbered List | `1. First`<br>`2. Second`              | 1. First<br>2. Second        |
| Link          | `[Google](https://google.com)`         | [Google](https://google.com) |
| Image         | `![alt](image.png)`                    | ![alt](image.png)            |
| Inline code   | `` `code` ``                           | `code`                       |
| Code block    | <pre>`python<br>print("Hi")<br>`</pre> |                              |

## **4. Example: README.md**

```markdown
# Pizza Project

Welcome to my pizza order app!

## Features

- Easy to use
- Custom toppings
- Fast delivery

## Getting Started

1. Clone the repo.
2. Run `python app.py`
3. Enjoy your pizza!

## Screenshot

![App screenshot](images/pizza.png)

> "Code is like pizza. The best is homemade."

```

## **5. Real-World Uses of Markdown**

* **README.md** in every GitHub repo (project overview, setup, usage).
* **Lab reports** or documentation for coding assignments.
* **Wiki pages** in class or work projects.
* **Note-taking:** Many modern note apps (Obsidian, Notion) use Markdown.
* **Blogging:** Many static blog generators use Markdown.
